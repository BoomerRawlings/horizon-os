# Horizon Workspace Instructions

This is a plain Markdown workspace used by Horizon. Keep it small, searchable, and easy to understand.

## Navigation

Start at `00_Index.md`, then use the nearest folder `index.md`.

## Calendar

- Store one dated item per Markdown file in `Calendar/Items/`.
- Do not invent dates or times. Use `unknown` when needed.
- Keep the next action clear enough to understand without reopening the original capture.

## Capture

- Raw captures go to `Inbox/Captures/`.
- Items waiting to be sorted go to `Inbox/To Triage/`.
- Parsing requests go to `Runs/CaptureQueue/`.
- Handle useful content before deleting or moving a raw capture.

## Safety

- Keep passwords and API keys out of Markdown files.
- Prefer updating an existing index over adding a new organizing system.
- Do not overwrite or delete user-authored files unless the user clearly requests it.
